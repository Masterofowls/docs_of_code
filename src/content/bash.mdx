<h2 id="functions-and-arrays">Functions and Arrays</h2>

```bash
#!/usr/bin/env bash
set -euo pipefail

sum() {
  local total=0
  for n in "$@"; do
    total=$((total + n))
  done
  printf '%s\n' "$total"
}

sum 4 8 15 16
```

<h2 id="testing">Testing and Debugging</h2>

- Validate syntax first with bash -n.
- Use shellcheck for static checks and common pitfalls.
- Run scripts with set -x in debug sessions to trace execution.
- Check exit codes and fail early on command errors.

<h2 id="common-errors">Common Error Patterns</h2>

- Unquoted variables causing word splitting.
- Missing set -euo pipefail in automation scripts.
- Silent failures hidden in multi-stage pipelines.
- Overwriting files with redirection when append is intended.

<h2 id="best-practices">Performance and Security</h2>

- Prefer explicit quoting and explicit path handling.
- Validate input before interpolation into commands.
- Keep scripts idempotent so reruns are safe.
- Move complex business logic into typed languages when needed.
