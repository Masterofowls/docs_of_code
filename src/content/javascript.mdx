const price = 1999;

  console.log('high-value order');
}

<h2 id="async">Asynchronous JavaScript</h2>

```js
async function loadUsers() {
  const response = await fetch('/api/users');
  if (!response.ok) throw new Error('Request failed');
  return response.json();
}

const result = await Promise.allSettled([loadUsers()]);
console.log(result[0].status);
```

<h2 id="arrays-objects">Arrays, Objects, and Iteration</h2>

- Prefer array methods for transformations: map, filter, reduce.
- Use for...of when you need early exits or clearer control flow.
- Use object spread for immutable updates.
- Prefer Map when keys are dynamic or non-string.

<h2 id="testing">Testing and Debugging</h2>

- Add unit tests for pure utilities and formatting/parsing helpers.
- Add integration tests for API adapters and async workflows.
- Use breakpoints and network inspection before patching.
- Capture failing input payloads so regressions become tests.

<h2 id="security">Performance and Security</h2>

- Never trust external payloads; validate with a schema at boundaries.
- Avoid eval-style execution and sanitize rendered user content.
- Profile first, then optimize hot paths with measured evidence.
