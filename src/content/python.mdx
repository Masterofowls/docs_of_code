<h2 id="testing">Testing and Debugging</h2>

- Keep tests small, deterministic, and data-driven.
- Reproduce with minimal input before patching logic.
- Use type hints and runtime validation at API boundaries.
- Use structured logging for request and pipeline tracing.

<h2 id="data-and-ml">Data and ML Progression</h2>

- NumPy and Pandas for arrays, tables, and data cleaning.
- Matplotlib for visualization and analysis output.
- ML flow: split data, fit model, validate metrics, repeat.

<h2 id="common-errors">Common Error Patterns</h2>

- Module import errors: activate the correct virtual environment.
- None handling bugs: validate optional values explicitly.
- Mutable default arguments: use None then initialize internally.

<h2 id="best-practices">Performance and Security</h2>

- Prefer readable code over clever one-liners in critical paths.
- Validate all external input and sanitize command boundaries.
- Audit dependencies and pin versions for reproducible builds.
