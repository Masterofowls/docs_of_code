# Python Field Guide

Python optimizes readability and expressiveness. Use it for APIs, data pipelines, automation, and ML tooling.

<h2 id="syntax">Syntax and Keywords</h2>

- Significant indentation defines code blocks.
- Use `match` for pattern matching in modern Python.
- Keep expressions explicit for maintainability.

```python
match status:
  case 200:
    print('ok')
  case _:
    print('retry')
```

<h2 id="imports-exports">Import and Export</h2>

- Use absolute imports inside packages.
- Export public API through `__init__.py` and `__all__`.

```python
# package/__init__.py
from .client import ApiClient
__all__ = ['ApiClient']
```

<h2 id="comments">Comments and Documentation</h2>

- Use docstrings for every public function/class.
- Prefer concise comments that explain intent, not syntax.

<h2 id="linting">Linting and Formatting</h2>

- Use `ruff` for linting and import hygiene.
- Use `black` for stable formatting.
- Enforce in CI with fail-fast behavior.

<h2 id="functions-reduction">Functions and Reduction</h2>

- Keep pure functions side-effect free.
- Use `functools.reduce` only when it improves clarity.

```python
from functools import reduce
nums = [1, 2, 3, 4]
total = reduce(lambda acc, x: acc + x, nums, 0)
```

<h2 id="async">Async Programming</h2>

- Use `asyncio` for high-latency I/O workloads.
- Avoid blocking calls in async functions.

```python
import asyncio

async def fetch_all(tasks):
  return await asyncio.gather(*tasks)
```

<h2 id="oop">Object Oriented Patterns</h2>

- Use `dataclass` for data carriers.
- Use protocols/ABC for contracts.

<h2 id="math">Math Methods</h2>

- `math` for scalar math, `statistics` for descriptive metrics.
- Use `decimal` for money-sensitive precision.

<h2 id="loops">Loops and Iteration</h2>

- Use `for item in iterable` as default.
- Prefer comprehensions for simple transforms.

<h2 id="booleans-logic">Booleans and Logic</h2>

- Prefer direct truthiness checks: `if users:`.
- Guard clauses reduce nested branches.

<h2 id="strings">Strings and Templates</h2>

- Prefer f-strings for readability.
- Use raw strings for regex: `r'\d+'`.

<h2 id="variables">Variables and Types</h2>

- Use `snake_case` names and type hints.
- Distinguish mutable vs immutable defaults in function signatures.

<h2 id="objects-arrays-dicts">Objects, Arrays, Dictionaries</h2>

- Core structures: lists, tuples, dicts, sets.
- Use dict comprehensions for normalization pipelines.

<h2 id="algorithms">Algorithms and Complexity</h2>

- Choose list for ordered scans, set/dict for constant-time lookups.
- Binary search on sorted arrays reaches $O(\log n)$.

<h2 id="commands">Commands and Tooling</h2>

```bash
python -m venv .venv
pip install -r requirements.txt
pytest -q
ruff check .
```

<h2 id="initialization">Initialization</h2>

- Start with `pyproject.toml`, virtual env, dependency lock.
- Add pre-commit hooks early.

<h2 id="project-structure">Project Structure</h2>

```text
src/
  app/
  domain/
  adapters/
tests/
```

<h2 id="testing">Testing</h2>

- Use `pytest` with parameterized tests.
- Keep fixtures small and composable.

<h2 id="debugging">Debugging</h2>

- Use `pdb` or VS Code breakpoints.
- Reproduce with minimal deterministic input first.

<h2 id="api-methods">API Methods and Contracts</h2>

- Validate request models using `pydantic`.
- Version routes (`/v1/`) and keep response envelopes stable.

<h2 id="logging">Logging and Monitoring</h2>

- Use structured JSON logs.
- Include `request_id`, user context, latency.

<h2 id="date-methods">Date and Time Methods</h2>

- Prefer timezone-aware values (`datetime.now(timezone.utc)`).
- Store UTC; convert only at boundaries.

<h2 id="common-errors">Common Error Tips</h2>

- `ModuleNotFoundError`: verify venv and import path.
- `TypeError: NoneType`: guard nullable values.
- Async deadlocks: avoid nested event loops.

<h2 id="best-practices">Performance and Security</h2>

- Cache expensive deterministic computations.
- Never trust input; validate and sanitize all external data.
- Keep dependencies patched and audited.
