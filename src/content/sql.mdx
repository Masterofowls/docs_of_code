<h2 id="learning-path">Learning Path from Fetched Topics</h2>

- Query basics: SELECT, WHERE, ORDER BY, LIMIT or TOP.
- Data changes: INSERT, UPDATE, DELETE.
- Relational joins: inner, left, right, full, self joins.
- Aggregation: GROUP BY, HAVING, COUNT, SUM, AVG.
- Schema and constraints: primary key, foreign key, indexes.
- Reliability and security: transactions, parameters, least privilege.

<h2 id="core-queries">Core Query Patterns</h2>

```sql
SELECT customer_id, COUNT(*) AS orders, SUM(total_amount) AS total_revenue
FROM orders
WHERE status = 'completed'
GROUP BY customer_id
HAVING SUM(total_amount) > 500
ORDER BY total_revenue DESC;
```

<h2 id="transactions">Transactions and Consistency</h2>

```sql
BEGIN;

UPDATE accounts
SET balance = balance - 100
WHERE id = 'A';

UPDATE accounts
SET balance = balance + 100
WHERE id = 'B';

COMMIT;
```

<h2 id="testing">Testing and Debugging</h2>

- Verify every migration forward and rollback path.
- Capture exact SQL with bound values when reproducing issues.
- Use explain analyze on slow production-class queries.
- Track lock wait and deadlock events.

<h2 id="security">Security and Performance</h2>

- Use prepared statements for all user-supplied parameters.
- Grant minimal permissions per service role.
- Add indexes from observed query patterns, then re-measure.
- Keep hot filters on typed indexed columns, not free-form text blobs.
